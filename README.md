Write up:

Database 
Pros: can hold large amounts of data, can be easily grown if needed, can run querries
Cons: Somtimes overly complicated, more complicated to setup, more expensive

Binary serialization
Pros: efficient and compact, securly store data in non-readable format
Cons: diffuclt to debug, more complex, not human readable

JSON
Pros: easy implementation, easy to read/debug good for small structures
Cons: larger file size, can be modified by players

I chose JSON becasue it is easy to implement, it is great for smaller projects that do
not have a lot of save data. I am not super concerened about the cons. For my game the 
larger file size is not a problem and I am not concerned about players modifying it.
